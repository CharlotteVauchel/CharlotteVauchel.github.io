Projets professionels

Outil de remplissage de base de données

Pour ce projet j’ai dû dans un premier temps créer une base de données sql Serveur via Microsoft Sql Server management studio. La problématique était de pouvoir entrer des informations sous diverses formes (excel, txt) et de les normaliser dans une base de données. Pour ce projet je suis partie sur la création d’une application console en C# permettant de garder une trace du remplissage de la base de données. L’application parcourt tous les fichiers importants et trie les informations à garder ou non. Je créée des objets intermédiaires pour garder en mémoire les informations et ne pas avoir de doublons ou des informations erronées dans la base de données. S’il y a une erreur dans les fichiers de collecte il y aura une erreur affichée dans la console pour permettre de régler le problème mais le programme continuera de tourner jusqu’à l’ajout des informations dans la base de données. Le remplissage de la base de données ne se fait qu’à la fin du programme, c’est la partie la plus rapide puisque la base de données a été créée préalablement, tous les problèmes de doublons ayant été géré auparavant il ne doit pas y avoir de problème au remplissage sauf en cas de problème avec la connexion à la base de données.

2022

Application de recherche dans la base de données

Dans la continuité de ce projet j'ai créé une application permettant d'aller rechercher les informations dans la base de données. Cette application a pour but de permettre à l'utilisateur de rechercher des informations dans la base de données, sans avoir à connaître le langage SQL, et donc compréhensible par tous.

Cette application va dans un d'abord charger toutes les informations de la base de données et afficher dans un menu toutes les informatiuons possibles. Il sera ensuite possible, via l'application, d'effectuer un tri, une recherche, selon des critères que l'on peut définir ou non. Ainsi, le menu n'affichera plus que les informations trié. Via cette application on peut ouvrir le répertoire dans l'explorateur de fichier qui nous ramène à l'endroit où se trouve le fichier avec les informations d'origine. On peut également exporter dans un fichier pdf la liste des informations triées avec la recherche effectuée. Lorsque l'on clique sur une information dans le menu on ouvre les détails sur celle-ci.

L'application possède un menu permettant de changer directement les informations du fichier de configuration, comme le nom de la base de données, l'adresse de celle-ci, le chemin des fichiers d'origine. Dans ce menu on retrouve également une fenêtre pour lancer l'application console créée pour remplir la base de données, ainsi que le menu pour modifier sa configuration.

Cette application m'a permit de découvrir le modèle MVVM (Modèle-Vue-Vue Modèle) mais également d'utiliser énormément de bibliothèques pour la création de l'interface graphique ou pour la création du fichier pdf. J'ai également prit en main l'utilisation et les avantages du fichier de configuration pour stocker des informations qui ne doivent pas être modifiées directement dans le code, et peuvent être modifiées sans avoir à de nouveau compiler le projet.

Module web

Dans le cadre d'un projet différent j'ai créé un module web affichant une courbe dont les données sont mise à jour en temps réel. Pour ce projet j'ai utilisé le framework ASP .NET Core en C# pour la partie back-end et D3.js pour la partie front-end. C'est un projet qui a nécessité de la réflexion pour la mise en place de la communication entre le back-end et le front-end. D3.js est une bibliothèque JavaScript qui permet de créer des graphiques dynamiques et interactifs sur une page web. Cette bibliothèque est très puissante et permet de créer des graphiques très complexes. J'ai pui créer deux curseurs mobiles avec des tolltips visible lorsqu'ils sont déplacé. La courbe peut également être zoomée.
